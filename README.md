# 资产扫码盘点工具

一个适合手机现场使用的资产盘点网页工具，支持扫码、拍照 OCR 识别、资产信息录入、编辑、导出 Excel。

## 在线使用

部署到 GitHub Pages 后，可直接用手机浏览器打开。建议在手机浏览器中“添加到主屏幕”，方便现场使用。

## 主要功能

- 扫描二维码
- 扫描一维条形码（Code 128、Code 39、EAN、UPC 等）
- 拍照并通过 OCR 尝试识别标签中的 SN 序列号和型号
- 手动填写型号、位置、使用人、状态和备注
- 已录入资产支持编辑、删除
- 自动保存在浏览器本地，关闭页面后通常仍可继续录入
- 一键导出 Excel
- 可自定义导出的 Excel 文件名

## 使用流程

1. 输入本次盘点的 Excel 文件名，例如：`显示器盘点`
2. 选择扫描方式：
   - 资产贴纸是二维码：选择“二维码”
   - 资产贴纸是一维长条码：选择“一维条形码”
3. 扫描 SN，或拍照后点击 OCR 识别
4. 补充型号、位置、使用人、状态、备注等信息
5. 点击“添加到列表”
6. 发现录入错误时，在列表中点击“编辑”，修改后保存
7. 完成盘点后点击“导出 Excel”

## 注意事项

- 首次使用扫码和 OCR 时，需要允许浏览器访问摄像头。
- 扫码功能建议在 HTTPS 页面中使用，例如 GitHub Pages。
- 条形码扫描时，尽量让条码横向占满取景框，避免反光和模糊。
- OCR 识别结果需要人工确认，尤其是 `O/0`、`I/1`、`S/5` 等容易混淆的字符。
- 浏览器本地数据依赖当前浏览器和设备；导出 Excel 后建议及时备份。
- 不要把包含公司内部资产信息的导出文件上传到公开仓库或公开网盘。

## 项目文件

```text
index.html   页面结构
style.css    页面样式
app.js       扫码、OCR、列表编辑、本地保存、Excel 导出逻辑
```

## 使用的第三方库

- [html5-qrcode](https://github.com/mebjas/html5-qrcode)：二维码和条形码扫描
- [Tesseract.js](https://github.com/naptha/tesseract.js)：图片 OCR 识别
- [SheetJS / xlsx](https://sheetjs.com/)：导出 Excel
